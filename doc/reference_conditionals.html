Conditionals

Conditional expressions allow DOM elements to be appear and disappear based on the value of properties in the model. Just like with binding, the behaviour is immediate when the properties in the model are changed by the controller. The attributes which control the hiding and showing of elements:

  • b:test="js expression" - Defines an expression that is evaluated to determine whether the element should be hidden. It is usually used inside the convenience element <b:if></b:if> as in:

            <b:if test="balance < 0" >Please deposit some cash</b:if>.
            (You don't need the b: namespace when BINDster attributes are used within BINDSter tags)

  • b:hide="hide method"determines the method for hiding and showing elements. These values are available:

    • remove - Hide by removing the element's children and moving them into a document fragment and copying them back into the DOM when the element is again shown. This is the default method for test.
    • display - Hide using display: none such that the element occupies no space. The old display value is remembered when the element is shown.
    • visibility - Hide using visibility: hidden such that the element occupies space but does not display. visiblity is set to visible when the element is shown.
    • offscreen - Hide by positioning the element absolute with a top of -32767px such that the element is properly rendered but not actually viewable. This is useful in some circumstances where you would want to use display: none but where you need the HTML to be fully rendered so heights and widths are calculated correctly even while hidden.

  • b:showif="js expression" - Short for test with b:hide="display" as in

            <span b:showif="balance < 0" >Please deposit some cash</span>

  • b:visibleif="js expression" - Short for test with b:hide="visibility" as in

            <span b:visibleif="balance < 0" >Please deposit some cash</span>
